규격 정의의 목적

규격을 정의하여 여러 곳에서 쉽게 가져다가 사용할 수 있게 합니다.


규격 규칙

기존항목을 삭제하지 않습니다. (이미 사용하던 곳에서 오류방지)

기존항목을 변경을 하지 않고 유동적으로 사용가능하도록 설계합니다.


라이센스

출처표기 : ncov-korea.com 주소와 주소링크를 포함하여 표기해야됨.

표기 위치는 자료 사용위치 주변에 위치해야되며, 이외에는 자유롭게 사용가능합니다.


API 호출

확진자 데이터 : https://ncov-korea.com/data.json로 데이터 파일에 접근이 가능합니다.

모두에게 개방되어있으며, CORS제한도 없습니다.


확진자 데이터 형식

    {
  • "info":{
    • "update":"2020-02-02 21:24", => 업데이트 시각
    • "version":"0.0.2", => 형식 버전
    • "Symptom_quarantine":174, => 유증상자(격리중)
    • "Symptom_release":522 => 유증상자(격리해제)
    },
  • "data":[
      {
    • "num":1, => 첫번째 감염자
    • "sex":"w", => 성별 : 여성, 남성은 m {"m","w"}
    • "age":35, => 나이 : 35세
    • "parent":0, => 모 감염자 번호 (기본값은 0)
    • "country":"cn", => cn: 중국, kr: 한국
    • "contact":45, => 접촉자 수
    • "hospital":"인천의료원", => 격리 병원
    • "from":"우한 방문", => 모감염자와 관계 또는 방문 지역 {"#0 지인","?? 방문", ...}
    • "wuhan":true, => 우한 방문여부
    • "status":1, => 상태 {0:사망, 1:격리 중, 2:완치}
    • "confirmDate":"2020-01-20" => 확진 날짜 {yyyy-mm-dd}
    • "dischargeDate":"2020-02-06" => 완치 날짜 {yyyy-mm-dd}
      • }
    ]
    • }
